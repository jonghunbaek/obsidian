서비스가 느려졌다면 해결을 위한 실마리를 어디서부터 찾아야 할까?
서버 성능과 관련된 중요한 두 가지 지표가 있다.
- **응답 시간**
- **처리량**
이 두 가지를 토대로 어디서 병목이 일어나는지 그 실마리를 찾아볼 수 있다.
### 응답 시간
응답 시간은 `TTFB(Time to First Byte)`와 `TTLB(Time to Last Byte)` 두 가지로 나누어 측정할 수 있다.
-> 첫 패킷 도착 시간까지 측정하냐, 마지막 패킷 도착 시간까지 측정하냐로 나눌 수 있음
이 지표중 상황에 맞는 지표로 측정하는 것이 바람직하다.

클라이언트에서 `서버와 연결하는 과정` 과 `데이터를 전송하는 과정` 크게 두 가지로 구분할 수 있다.
서버와 연결하는 과정은 TCP 3way-handshake라 불리는 과정이며, 데이터를 전송하는 과정에선 다음과 같은 작업이 발생한다.
- 로직 수행 - Low
- DB 연동 - High
- 외부 API 연동 - High
- 응답 데이터 생성 - Low
DB, 외부 API 연동 부분이 성능 개선의 핵심이 되는 경우가 많다.
