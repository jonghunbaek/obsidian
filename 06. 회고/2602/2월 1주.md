# 1. 업무
### 1월말 발생한 XSS 공격 수습
- 최초 공격 인지는 얻어 걸린 슬랙 장애 아림
- 최초엔 어떤 방식으로 공격이 오는지도 이해를 못함
- 1차적으로 게시판에 HTML 코드를 검증하는 XSS 방어로직 개발
- 다양한 유스케이스에 대응하여 방어해야 하므로 생각보다 복잡해짐
- 핵심은 UX를 전혀 떨어뜨리지 않으면서 검증해야 하는 것
### 2차 XSS 공격 수습
- 공격은 크게 게시글에 스크립트 포함 + 파일 업로드 시 파일 확장자명 변조 및 파일에 스크립트 삽입
- 이미지 파일에 스크립트를 심어 공격하는 Polyglot 
- 이에 대한 해결책이 몇 가지 있으나, 현상황에선 굳이 안해도 될거라 판단
	- 구버전 브라우저의 경우 문제가 될 수 있음
	- 이미지 파일에 심어진 스크립트를 실행하는 것이 일반 사용자에겐 허들
- 파일명, 확장자, mime 타입, 매직 넘버 검증만 수행
### oas 파이프라인
- example별도 분리
	- example 크기가 너무 큰 경우들이 존재함
- github action을 활용한 자동화 워크플로우 추가
